# Главная идея QoS

Когда канал загружен:

- пакеты начинают стоять в очереди;
- появляются задержки;
- теряются пакеты;
- голос лагает;
- видео зависает;
- сайты тормозят.

QoS нужен, чтобы:

- важный трафик проходил первым;
- менее важный мог подождать.

# Что вы сделали в этой лабораторной

Вы:

1. нашли определённый трафик;
2. классифицировали его;
3. пометили DSCP/COS метками;
4. подготовили сеть к приоритизации.

Это называется:

# Classification + Marking

---

# Что происходит по шагам

---

# 1. ACL — поиск нужного трафика

Например:

```
ip access-list extended ACL-WEB permit tcp any any eq 80 permit tcp any any eq 443
```

Это не фильтрация.

ACL здесь используется как:

# «поисковый фильтр»

Router говорит:

> "Ага, это HTTP/HTTPS трафик."

---

# 2. class-map — создание класса трафика

```
class-map match-any CM-WEB match access-group name ACL-WEB
```

Теперь Cisco создаёт логическую группу:

# CM-WEB

Туда попадают все web-пакеты.

---

# 3. policy-map — что делать с этим трафиком

```
class CM-WEB set dscp af43
```

Теперь роутер:

- не ограничивает;
- не дропает;
- пока только маркирует.

То есть:

# «Этот трафик важный.»

---

# Что такое DSCP

В IP-пакете есть специальное поле:

# DSCP (Differentiated Services Code Point)

Оно находится в IP header.

Маршрутизаторы читают это поле и понимают:

- кому дать приоритет;
- кого обслужить быстрее;
- кого можно задержать.

---

# Пример из жизни

Представьте:

- VoIP звонок
- YouTube
- Backup 100 ГБ

Без QoS:

- всё смешивается;
- голос начинает хрипеть.

С QoS:

- VoIP получает приоритет;
- backup ждёт.

---

# Почему SIP пометили CS5

```
set dscp cs5
```

SIP — это сигнализация VoIP.

Телефонии нужны:

- минимальная задержка;
- стабильность.

Поэтому:

# SIP всегда высокоприоритетный.

---

# Почему HTTP/HTTPS AF43

AF43 — это:

# Assured Forwarding

То есть:

> "Желательно доставить хорошо."

Web-трафик:

- важен;
- но не критичен как голос.

---

# Почему ICMP CS3

ICMP:

- ping;
- traceroute;
- диагностика.

Если сеть перегружена:

- ICMP не должен убивать голос;
- но и полностью теряться не должен.

Поэтому:

# средний приоритет.

---

# Почему EF для Client2/Client3 → Client1

```
set dscp ef
```

EF = Expedited Forwarding.

Это:

# самый высокий приоритет.

Используется для:

- VoIP RTP;
- real-time traffic;
- критичных приложений.

В вашем задании:

- трафик к Client1 считается критичным.

---

# Почему policy применяли на Gi0/1

```
interface Gi0/1 service-policy input QOS-MARKING
```

Gi0/1 смотрит в LAN.

Трафик:

- приходит от клиента;
- сразу маркируется;
- дальше идёт через всю сеть уже с DSCP.

Это называется:

# edge marking

Лучше маркировать:

- максимально близко к источнику.

---

# Что происходит дальше в сети

После маркировки другие устройства могут:

---

# Queueing

Например:

|Очередь|Трафик|
|---|---|
|Priority Queue|EF|
|High Queue|CS5|
|Medium Queue|AF43|
|Default Queue|обычный|

---

# Congestion Management

При перегрузке:

- EF пойдёт первым;
- AF43 позже;
- default может дропаться.
