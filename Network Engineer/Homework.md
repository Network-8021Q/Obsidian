
Тема: Проектирование WAN сегмента

Задача:

1. Сделать отказоустойчивый шлюз в центральном офисе → HSRP.
2. Соединить офисы через Интернет → GRE.
3. Автоматически обмениваться маршрутами → OSPF.

---

# 1. HSRP (Hot Standby Router Protocol)

## Зачем нужен

Если Router1 сломается, компьютеры не должны потерять связь.

Без HSRP:

```
PC → Router1 → Интернет
```

Если Router1 выключился:

```
PC ✖ Router1
```

Связь пропала.

---

## Что делает HSRP

Два роутера создают виртуальный шлюз.

Например:

```
Router1 = 192.168.10.2Router0 = 192.168.10.3Виртуальный IP = 192.168.10.1
```

ПК используют:

```
Gateway = 192.168.10.1
```

---

### Роли

#### Active

Главный роутер

```
Router1
```

Отвечает за трафик.

#### Standby

Резервный роутер

```
Router0
```

Ждёт отказа Active.

---

### Команды

```
standby 1 ip 192.168.10.1
```

Создает виртуальный шлюз.

---

```
standby 1 priority 110
```

Чем больше число, тем выше шанс стать Active.

---

```
standby 1 preempt
```

Если основной вернулся после аварии — снова становится Active.

---

## Проверка

```
show standby brief
```

Увидишь:

```
Router1 = ActiveRouter0 = Standby
```

---

# 2. GRE (Generic Routing Encapsulation)

## Проблема

Router1 знает только Интернет:

```
100.0.0.0/30
```

Router2 знает только:

```
102.0.0.0/30
```

Локальные сети:

```
192.168.10.0/24192.168.20.0/24
```

через Интернет не передаются.

---

## Что делает GRE

Создает виртуальный кабель через Интернет.

Было:

```
Router1 ---- Internet ---- Router2
```

Стало:

```
Router1 ==== Tunnel ==== Router2
```

---

### Tunnel интерфейсы

На Router1:

```
interface Tunnel0 ip address 10.10.10.1 255.255.255.252
```

На Router2:

```
interface Tunnel0 ip address 10.10.10.2 255.255.255.252
```

Получилась отдельная сеть:

```
10.10.10.0/30
```

---

### Tunnel Source

Откуда отправлять GRE.

У тебя:

```
tunnel source GigabitEthernet0/1
```

То есть:

```
100.0.0.2
```

---

### Tunnel Destination

Куда отправлять GRE.

```
tunnel destination 102.0.0.2
```

То есть:

```
Router2
```

---

### Почему была проблема

У тебя отсутствовал:

```
tunnel source
```

На Router2.

Получалось:

```
Туннель знает куда идти,но не знает откуда отправлять пакеты.
```

Поэтому OSPF не работал.

---

# 3. OSPF (Open Shortest Path First)

## Зачем нужен

Без OSPF пришлось бы вручную писать маршруты.

Например:

На Router1:

```
ip route 192.168.20.0 255.255.255.0 ...
```

На Router2:

```
ip route 192.168.10.0 255.255.255.0 ...
```

Это неудобно.

---

## Что делает OSPF

Роутеры автоматически обмениваются маршрутами.

---

### Соседство (Neighbor)

Сначала роутеры знакомятся.

Через GRE:

```
Router1 (10.10.10.1)        |      OSPF        |Router2 (10.10.10.2)
```

---

### Hello пакеты

Каждые несколько секунд:

```
Привет!Я Router1.
```

---

```
Привет!Я Router2.
```

---

После этого образуется соседство:

```
FULL
```

Проверка:

```
show ip ospf neighbor
```

---

### Обмен маршрутами

Router1 сообщает:

```
У меня есть сеть:192.168.10.0/24
```

Router2 сообщает:

```
У меня есть сеть:192.168.20.0/24
```

---

После этого:

На Router1:

```
O 192.168.20.0/24
```

На Router2:

```
O 192.168.10.0/24
```

Буква:

```
O
```

означает:

```
Получено через OSPF
```

---

# Почему ПК сначала не пинговались

Последовательность ошибок была такой:

### Ошибка 1

На Router6 не был настроен:

```
102.0.0.1
```

Поэтому Router1 не видел Router2.

---

### Ошибка 2

Использовались switch-порты:

```
Gi0/1/x
```

На них нельзя назначать IP.

---

### Ошибка 3

На Router2 отсутствовал:

```
tunnel source
```

Поэтому GRE не мог нормально передавать трафик.

---

### Ошибка 4

Из-за GRE не работал OSPF.

Поэтому не появлялись маршруты:

```
O 192.168.10.0/24O 192.168.20.0/24
```

---

# Как теперь проходит пакет

Допустим:

```
PC0192.168.10.10
```

пингует:

```
PC3192.168.20.10
```

Путь:

```
PC0 ↓192.168.10.1 (HSRP) ↓Router1 (Active) ↓GRE Tunnel ↓Router2 ↓192.168.20.10
```

Если Router1 выключить:

```
Router0 становится Active
```

И путь будет:

```
PC0 ↓192.168.10.1 ↓Router0 ↓GRE Tunnel ↓Router2 ↓PC3
```

Связь сохранится.

---

# Что спрашивают преподаватели чаще всего

### Что такое HSRP?

Протокол отказоустойчивости шлюза. Создает виртуальный IP, который используют клиенты как gateway.

### Что такое GRE?

Туннельный протокол Cisco, создающий виртуальный канал через IP-сеть.

### Что такое OSPF?

Динамический протокол маршрутизации, который автоматически обменивается маршрутами между роутерами.

### Почему OSPF запустили через GRE?

Потому что OSPF — это не просто IP-трафик. Через обычный Интернет он обычно не строится между удаленными офисами. GRE создает логический канал, поверх которого OSPF может обмениваться маршрутами.

Это уже полноценная корпоративная схема: **HSRP для отказоустойчивости, GRE для связи филиалов и OSPF для автоматической маршрутизации**.
