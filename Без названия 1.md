# Задание 2. HSRP

На Router0:

```
interface g0/0 ip address 192.168.10.2 255.255.255.0 standby 1 ip 192.168.10.1 standby 1 priority 110 standby 1 preempt
```

На Router1:

```
interface g0/0 ip address 192.168.10.3 255.255.255.0 standby 1 ip 192.168.10.1 standby 1 priority 100 standby 1 preempt
```

Результат:

- Router0 Active
- Router1 Standby
- при отказе Router0 шлюз остается доступным.

# Краткое описание решения

### Отказоустойчивость шлюза

Использован HSRP:

- виртуальный шлюз 192.168.10.1;
- Router0 активный;
- Router1 резервный;
- автоматическое переключение при отказе.

### Связь между офисами

Созданы два независимых GRE-туннеля:

1. Router0 ↔ Router2
2. Router1 ↔ Router2

Это позволяет использовать оба интернет-канала центрального офиса.

### Маршрутизация

Использован OSPF Area 0.

OSPF автоматически:

- обменивается маршрутами между офисами;
- обнаруживает отказ туннеля;
- перестраивает таблицы маршрутизации;
- обеспечивает резервирование через второй GRE.

Такое решение соответствует типовой корпоративной схеме «Dual-Hub GRE + HSRP + OSPF».
