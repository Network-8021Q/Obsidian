# СХЕМА

## R1

- Gi0/0 → 100.0.0.1
- Gi0/1 → 192.168.5.1

## R2

- Gi0/0 → 100.0.0.2
- Gi0/1 → 172.16.1.1

## vIOS

- Gi0/0 → 100.0.0.3

# =========================

# НАСТРОЙКА R2

# =========================

Подключаемся к R2.

---

## 1. Переходим в конфигурацию

```
enableconfigure terminal
```

---

# 2. Создаём ACL

---

## HTTP/HTTPS

```
ip access-list extended ACL-WEB permit tcp any any eq 80 permit tcp any any eq 443 exit
```

---

## SIP

```
ip access-list extended ACL-SIP permit udp any any range 5060 5061 exit
```

---

## ICMP

```
ip access-list extended ACL-ICMP permit icmp any any exit
```

---

## EF для Client2 → Client1

```
ip access-list extended ACL-EF permit ip host 172.16.1.20 host 192.168.5.10 exit
```

---

# 3. Создаём class-map

---

## WEB

```
class-map match-any CM-WEB match access-group name ACL-WEB exit
```

---

## SIP

```
class-map match-any CM-SIP match access-group name ACL-SIP exit
```

---

## ICMP

```
class-map match-any CM-ICMP match access-group name ACL-ICMP exit
```

---

## EF

```
class-map match-any CM-EF match access-group name ACL-EF exit
```

---

# 4. Создаём policy-map

```
policy-map QOS-MARKING
```

---

## EF

```
 class CM-EF  set dscp ef exit
```

---

## SIP

```
 class CM-SIP  set dscp cs5 exit
```

---

## WEB

```
 class CM-WEB  set dscp af43 exit
```

---

## ICMP

```
 class CM-ICMP  set dscp cs3 exit
```

---

## default

```
 class class-default  set dscp default exit
```

---

## Выход

```
exit
```

---

# 5. Применяем policy на интерфейс LAN

Трафик приходит от клиента через Gi0/1.

```
interface GigabitEthernet0/1 service-policy input QOS-MARKING exit
```

---

# 6. Сохраняем

```
endwrite memory
```

---

# =========================

# НАСТРОЙКА vIOS

# =========================

Настройка почти такая же.

---

## 1. Конфигурационный режим

```
enableconfigure terminal
```

---

# 2. ACL

---

## WEB

```
ip access-list extended ACL-WEB permit tcp any any eq 80 permit tcp any any eq 443 exit
```

---

## SIP

```
ip access-list extended ACL-SIP permit udp any any range 5060 5061 exit
```

---

## ICMP

```
ip access-list extended ACL-ICMP permit icmp any any exit
```

---

## EF для Client3 → Client1

```
ip access-list extended ACL-EF permit ip host 10.10.10.30 host 192.168.5.10 exit
```

---

# 3. class-map

```
class-map match-any CM-WEB match access-group name ACL-WEB exitclass-map match-any CM-SIP match access-group name ACL-SIP exitclass-map match-any CM-ICMP match access-group name ACL-ICMP exitclass-map match-any CM-EF match access-group name ACL-EF exit
```

---

# 4. policy-map

```
policy-map QOS-MARKING class CM-EF  set dscp ef exit class CM-SIP  set dscp cs5 exit class CM-WEB  set dscp af43 exit class CM-ICMP  set dscp cs3 exit class class-default  set dscp default exitexit
```

---

# 5. Применяем policy

Трафик приходит от Client3 через Gi0/1.

```
interface GigabitEthernet0/1 service-policy input QOS-MARKING exit
```

---

# 6. Сохраняем

```
endwrite memory
```

# ВАЖНЫЙ МОМЕНТ

Порядок class в policy-map важен.

Cisco проверяет сверху вниз.

Поэтому:

1. EF должен идти первым
2. Потом SIP
3. Потом WEB
4. Потом ICMP

Иначе пакет может попасть не в тот class.
