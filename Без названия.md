# 3. WAN-соединения между офисами

Для соединения маршрутизаторов используем сети /30.

|Соединение|Сеть|
|---|---|
|Central ↔ Office1|172.16.1.0/30|
|Central ↔ Office2|172.16.2.0/30|
|Central ↔ Office3|172.16.3.0/30|

Пример:

### Central ↔ Office1

- Central-R = 172.16.1.1
- Office1-R = 172.16.1.2

# 5. Маршрутизация

Для лабораторной работы удобно использовать OSPF.

На всех маршрутизаторах:

```
router ospf 1 network 10.0.0.0 0.255.255.255 area 0 network 172.16.0.0 0.0.255.255 area 0
```

После настройки все офисы автоматически обменяются маршрутами.

# 6. Проверка работы

Необходимо проверить:

1. ПК центрального офиса пингуют удалённые офисы.
2. ПК удалённых офисов пингуют серверы/устройства центрального офиса.
3. Есть связность между всеми маршрутизаторами.
4. VLAN изолированы на канальном уровне.
5. Межсетевое взаимодействие работает через маршрутизатор.

### 4. Красные треугольники

На схеме видно много красных индикаторов возле интерфейсов маршрутизаторов.

Это означает, что интерфейсы выключены.

На каждом маршрутизаторе нужно выполнить:

```
conf tinterface g0/0no shutdowninterface g0/1no shutdowninterface g0/2no shutdown
```

После этого индикаторы станут зелёными.
