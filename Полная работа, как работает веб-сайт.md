
==Пользователь вводит URL в браузере==
Вы открываете браузер и вводите, например, в адресную строку.www.example.com

==Проверка кэша браузера==
Браузер сначала проверяет, нет ли уже сохранённой (закэшированной) версии этой страницы на вашем компьютере. Если есть - страница может быть загружена сразу.

==DNS-запрос==
Если страница не найдена в кэше, браузер отправляет запрос к DNS-серверу, чтобы узнать, какой IP-адрес соответствует введённому доменному имени ().www.example.com

==Получение IP-адреса от DNS==
DNS-сервер возвращает браузеру IP-адрес, по которому находится нужный сайт.

==Установка TCP-соединения с веб-сервером==
Браузер инициирует соединение с сервером по протоколу TCP (Transmission Control Protocol) по полученному IP-адресу, обычно на порт 80 (HTTP) или 443 (HTTPS).

==Отправка HTTP-запроса==
После установления соединения браузер отправляет HTTP-запрос (например, GET) на сервер, чтобы получить содержимое страницы.

==Обработка запроса сервером==
Веб-сервер получает запрос, обрабатывает его (например, ищет нужную страницу или запускает скрипты для генерации контента).

==Сервер отправляет HTTP-ответ==
Сервер формирует HTTP-ответ (например, с HTML-кодом страницы) и отправляет его обратно браузеру.

==Получение и рендеринг страницы браузером==
Браузер получает ответ, анализирует HTML, загружает дополнительные ресурсы (картинки, стили, скрипты) и отображает страницу пользователю.цвФьтмиьь3
