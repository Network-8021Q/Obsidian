# **==OWASP Top-10 — это ежегодно обновляемый рейтинг 10 самых критичных уязвимостей веб-приложений, составленный организацией _Open Web Application Security Project (OWASP)_. Это мировой стандарт для оценки рисков безопасности в IT.

1) ==**Инъекции (Injection)**== 
		- Атаки через SQL, NoSQL, OS-команды.
		- _Пример_: `' OR 1=1 --` в поле логина.
2) ==**Небезопасная аутентификация** (**Broken Authentication**)==
		- Слабые пароли, отсутствие 2FA, утечка сессий.
		- _Пример_: Брутфорс аккаунта `admin:admin`.
3) ==**Раскрытие конфиденциальных данных** (**Sensitive Data Exposure**)==
		- Утечка персональных данных, кредитных карт.
		- _Пример_: Незащищённая БД с паролями в открытом виде.
4) ==**Уязвимые компоненты** (**Vulnerable Components**)==
		- Использование библиотек с известными уязвимостями.
		- _Пример_: Устаревшая версия jQuery или Log4j.
5) ==**Неправильная настройка безопасности ** (**Security Misconfiguration**)==
		- Открытые порты, дефолтные учётные записи.
		- _Пример_: Доступ к `/admin` без пароля.
6) ==**Уязвимости контроля доступа** (**Broken Access Control**)==
		- Обход прав (IDOR, вертикальный/горизонтальный эскалации).
		- _Пример_: Подмена `user_id=100` → `user_id=101` в URL.
7) ==**Межсайтовый скриптинг (XSS)** (**Cross-Site Scripting (XSS)**)==
		- Внедрение вредоносного JS-кода.
		- _Пример_: `<script>alert(1)</script>` в поле комментария.
8) ==**Небезопасная десериализация** (**Insecure Deserialization**)==
		- RCE через поддельные сериализованные объекты.
		- _Пример_: Эксплойт в Apache Struts.
9) ==**Мониторинг и логирование** (**Insufficient Monitoring/Logging**)==
		- Отсутствие аудита атак.
		- _Пример_: Невозможность отследить брутфорс.
10) ==**Подделка межсайтовых запросов (CSRF)** (**CSRF (Cross-Site Request Forgery)**)==
		- Принуждение пользователя к действиям без согласия.
		- _Пример_: Скрытая форма для перевода денег.





- [Шаг 7 – Сканирование портов – Stepik](https://stepik.org/lesson/958312/step/7?unit=964756) - расширенный поиск портов, также множество протоколов 
- **==ICMP==** расшифровывается как **Internet Control Message Protocol** - протокол межсетевых управляющих сообщений
- ==ping== — это команда, которая отправляет пакет ICMP Echo в удаленную систему.
- команда ==traceroute== _отслеживает маршрут_, пройденный пакетами от вашей системы к другому узлу. Целью трассировки является поиск IP-адресов маршрутизаторов или узлов, через которые проходит пакет, когда он идет от вашей системы к целевому узлу. Эта команда также показывает количество маршрутизаторов между двумя системами.
- ==TTL( Time To Live==) - это количество времени или «скачков», в течение которого пакет должен существовать в сети, прежде чем он будет отброшен маршрутизатором
- ==Протокол TELNET (Teletype Network)== был разработан в 1969 году для связи с удаленной системой через интерфейс командной строки (CLI). Следовательно, команда использует протокол TELNET для удаленного администрирования. По умолчанию telnet использует ==порт 23.==
- ==Netcat== или просто имеет различные приложения, которые могут иметь большую ценность для пентестера. Netcat поддерживает протоколы TCP и UDP. Он может функционировать как клиент, который подключается к порту прослушивания;
- Найдите IP-адрес доменного имени с помощью , что расшифровывается как Name Server Look Up (поиск на сервере имен). Вам нужно ввести команду , например. Этими тремя основными параметрами являются: ==nslookup==
- ==Поддомен== — как отдельный магазин в торговом центре (основном сайте). Каждый магазин имеет своё название и ассортимент, но все они находятся в одном здании
- Протокол ==**SMB (Server Message Block)**== — это сетевой протокол прикладного уровня, предназначенный для удалённого доступа к файлам, принтерам и другим сетевым ресурсам, а также для межпроцессного взаимодействия в компьютерных сетях. ==445 порт==
- ==NETBIOS== - Он предоставляет услуги, связанные с сессионным уровнем модели OSI, позволяя приложениям на разных компьютерах взаимодействовать через локальную сеть. ==порт 139==
- ==Whois==, по сути, позволяет вам спрашивать, на кого зарегистрировано доменное имя. В Европе личные данные отредактированы; Тем не менее, в других местах вы потенциально можете получить большой объем информации из Whois искать.
- ==ARP-запрос== — это сетевое широковещательное сообщение, которое устройство в локальной сети отправляет для определения физического (MAC) адреса другого устройства по известному ему IP-адресу. ==ARP-запрос== — это механизм сопоставления IP-адреса с MAC-адресом в локальной сети, необходимый для корректной передачи данных
- ==**SSL (Secure Sockets Layer)** и **TLS (Transport Layer Security)**== — это криптографические протоколы, обеспечивающие безопасную передачу данных между клиентом и сервером.  Сейчас **TLS (версии 1.2 и 1.3)** является актуальным стандартом, а SSL устарел и считается небезопасным.
- ==**SMTP**== — это стандартный протокол для отправки электронной почты через интернет. Он работает на **порту 25 (без шифрования)**, **587 (с TLS)** или **465 (SMTPS, устарел)**.
- **==OAST==** — это метод тестирования безопасности, который выявляет уязвимости через **асинхронные (out-of-band) запросы** к внешнему серверу. Используется, когда классические методы (например, слепые SQL-инъекции) не дают прямого отклика.
- ==OSINT== - это методология сбора, анализа и использования открытой информации из различных источников для получения разведывательных данных или информации, полезной для принятия решения.
- ==**IDS== (Обнаружение**) и **==IPS== (Предотвращение**)— это системы обнаружения и предотвращения вторжений, которые защищают компьютерные сети от атак, аномалий и злонамеренной активности.
- ==OWA (Outlook Web Access)== - метод анализа сетевых протоколов, который позволяет получить информацию о том, какие пользователи и группы имеют доступ к OWA**
